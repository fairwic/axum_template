# Task Plan: 小卖部小程序后端 V1.0

## Goal
完成 V1.0 可上线的后端开发（先 Sprint 1 基础：门店/分类/商品/搜索/购物车 + 门店切换规则），并具备后续 Sprint 扩展能力。

## Phases
- [ ] Phase 1: 需求对齐与设计（含方案对比、数据模型与接口）
- [ ] Phase 2: 研究/梳理现有代码与规范（DDD 分层、错误/响应、SQLx 约定）
- [ ] Phase 3: 计划与实现（TDD + 逐模块落地）
- [ ] Phase 4: 验证与交付（测试/检查/文档）

## Key Questions
1. 配送超距是否允许加价配送？（已确认：允许）
2. 取消规则以哪条为准？（已确认：仅未支付取消）
3. 距离计算与地图服务使用哪家？（已确认：腾讯位置服务）

## Decisions Made
- Sprint 1 优先做门店/分类/商品/搜索/购物车与门店切换规则
- 微信登录；支付先做 mock
- 距离计算：腾讯位置服务 API
- 超距允许加价配送；按公里加价，距离向上取整到 1km
- 后台鉴权：管理员登录（手机号+密码），角色：平台/门店
- 会员默认全员会员
- 自动接单：Redis + 后台 worker
- 搜索：Postgres ILIKE
- 购物车：数据库存储
- 库存：下单即锁定
- 优惠券：仅占位
- 跑腿：固定费用
- 门店最近选择：前端本地保存
- 地址坐标：用户不手填，地图自动获取并存储
- 取消规则：仅未支付取消（与 PRD 原文不同）
